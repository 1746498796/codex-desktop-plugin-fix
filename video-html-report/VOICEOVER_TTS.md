# 适合 AI 朗读的口播稿

## 使用建议

- 不要让 TTS 直接读很长的 Windows 路径。
- 不要让 TTS 读太多英文插件名。
- 路径用画面展示，口播只说“这个 manifest 文件”。
- 一句话尽量 8 到 18 个字。
- 语速：1.05 到 1.12 倍。
- 音色：沉稳科技男声 / 新闻报告男声。

## 70 秒版

最近这两天，
我遇到一个挺典型的问题。

很多人的 Codex Desktop，
看不到浏览器插件，
也看不到电脑控制插件。

Chrome 调不出来。
Computer Use 也不能用。

一开始，
我也以为是浏览器扩展的问题。
或者是 API key 的问题。

但我结合 AI 一起排查之后，
发现真正的原因，
并不在这里。

问题出在 Codex 本地插件市场。

准确说，
是 openai bundled 这一组内置插件，
没有被 Codex 正确识别。

关键点是一个 manifest 文件。

它必须出现在指定目录里。

只放在根目录，
是不够的。

所以修复思路很简单。

第一步，
找到 Codex 安装包里，
官方自带的 bundled 插件。

第二步，
把它复制到用户目录下，
变成本地持久插件市场。

第三步，
补齐那个关键的 manifest 文件。

第四步，
在配置里启用浏览器插件，
Chrome 插件，
还有电脑控制插件。

然后重启 Codex。

修复之后，
Chrome 可以正常连接。
电脑控制也恢复了。

这不是破解，
也不是绕过限制。

只是让 Codex，
正确读取它安装包里，
本来就带着的官方插件。

我把这次排查过程，
整理成了文档和脚本。

现在已经放到 GitHub。

如果你也遇到类似问题，
可以按这个思路检查一下。
